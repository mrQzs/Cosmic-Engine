# CyberGeek — 宇宙主题博客概念设计文档

> **站点名称**: CyberGeek
> **副标题**: CyberCode Chronicles
> **域名**: wo.city
> **项目代号**: Cosmic-Engine（宇宙引擎）
> **设计理念**: 将博客系统抽象为一个完整的宇宙——用户不是在"浏览网页"，而是在"探索太空"。

---

## 一、核心世界观

### 1.1 一句话定义

> 每一篇文章是一颗行星，每一条评论是一颗卫星，每一个分类是一个星系，每一个标签是一片星云。你不是在阅读博客，你是在穿越宇宙。

### 1.2 设计哲学

- **视觉即信息**：天体的大小、颜色、亮度、轨道速度都编码了内容属性——无需阅读文字，一眼就能感知哪颗星球蕴含最密集的知识
- **交互即隐喻**：发表评论 = 发射卫星入轨；搜索 = 扫描太空；分类浏览 = 在星系间跃迁
- **演化即生长**：博客不是静态的，它会随内容积累自发演化——新子分类的恒星会"点燃"，热门文章的行星会吸引更多卫星形成壮观的行星环

## 二、宇宙天体层级

### 2.1 完整层级结构

```
宇宙 (Universe)
│
├── 脉冲星 (Pulsar) ─────── 个人简介，宇宙的绝对中心
│   └── 通信卫星 ─────────── 社交链接（GitHub/Twitter/邮箱等）
│
├── 星系 (Galaxy) ─────────── 主分类（如"杂志"、"论文"、"小说"）
│   │
│   ├── ● 超大质量黑洞 ────── 星系中心，引力锚点 + 分类概览入口
│   │     └── 吸积盘亮度 ∝ 分类活跃度（文章/评论越多越亮）
│   │
│   ├── ★ 恒星 (Star) ────── 子分类（如"科技"、"医学"、"经济"）
│   │   ├── 🌍 行星 (Planet) ── 文章
│   │   │   ├── 🌑 卫星 ─────── 评论
│   │   │   ├── 💫 光环 ─────── 高赞评论多时出现（类似土星环）
│   │   │   └── 🌑 卫星 ─────── 评论
│   │   └── 🌍 行星 ─────────── 文章
│   │
│   ├── ☁ 原恒星盘 ──────── 子分类（文章 < 10 篇，尚未"点燃"）
│   │   └── 🌍 行星（隐约可见）
│   │
│   └── 🌌 星云 (Nebula) ──── 标签云（标签越热门星云越亮）

├── ☄ 彗星 (Comet) ──────── 置顶文章（拖着尾巴在星系间穿行）
```

## 三、恒星生命周期（子分类的视觉演化）

这是整个宇宙中最独特的设计——子分类不是静态的标签，而是会随内容积累不断演化的恒星。

### 3.1 四个生命阶段

| 阶段         | 触发条件     | 视觉                                             | 隐喻                           |
| ------------ | ------------ | ------------------------------------------------ | ------------------------------ |
| **原恒星盘** | 文章数 < 10  | 暗淡的旋转气体云团，行星隐约可见                 | 知识正在积累，尚未成体系       |
| **主序星**   | 文章数 ≥ 10  | 明亮的自发光球体，暖黄色，表面对流纹理，日冕光晕 | 知识领域形成，开始照亮周围     |
| **巨星**     | 文章数 ≥ 50  | 体积 ×1.5，偏橙色，对流更剧烈                    | 深厚积累，这个领域已经成为专长 |
| **红巨星**   | 文章数 ≥ 100 | 体积 ×2.5，深红色，抛射星风粒子                  | 最厚重的知识领域，影响深远     |

### 3.2 生命周期可逆

恒星生命周期是**可逆**的——当文章数降回阈值以下时，恒星退回上一阶段（"熄灭"）。页面加载直接呈现最新状态，不模拟"变化过程"。只有用户主动触发的交互才需要动画。

### 3.3 恒星诞生动画

当子分类的文章数从 9 增长到 10——触发"恒星点燃"：

1. 原恒星盘的粒子加速旋转、向中心收缩
2. 中心开始发出微弱白光
3. **临界点**——白光爆发，冲击波环形扩散
4. 残余云雾被吹散，一颗明亮的主序星出现
5. 之前隐约可见的行星，现在清晰围绕新生恒星公转

## 四、星系中心超大质量黑洞

### 4.1 视觉设计

每个星系（主分类）中心都有一个超大质量黑洞——不再映射"加密文章"，而是星系本身的物理核心。

- **引力透镜效果**：Raymarching 光线步进，扭曲背后星空
- **吸积盘**：旋转的发光环带，颜色 = 星系主题色
- **视觉大小** ∝ 星系总文章数——文章越多，黑洞越壮观
- **吸积盘旋转速度** ∝ 近期发布频率——活跃的星系更有生命力

### 4.2 交互

点击黑洞 → 全息投影式概览面板展开在吸积盘上方：

- 星系名称和描述
- 恒星（子分类）列表及各自文章数
- 热门文章 Top 5
- 最近发布 5 篇
- 点击可直接跳转到对应恒星或行星

## 五、四大核心交互场景

### 5.1 宇宙全景探索

**首屏体验**：

- 深空背景 + 多个星系散布在三维空间
- 摄像机从全局俯瞰视角开始（固定视角）
- 星系根据文章数动态排布，新增星系触发重排（过渡动画让用户看到"搬家"过程）
- 首次加载范围：1.5 倍可视范围
- 右下角小地图、边缘罗盘指示器、面包屑导航

**操控方式**：

- 鼠标拖拽旋转 + 滚轮缩放（Google Earth 风格）
- 点击天体 → 摄像机 3~5 秒平滑飞行至目标
- ESC 逐层返回（阅读 → 行星 → 星系 → 全景）
- Tab 快速跳转菜单

**渐进式揭示**：

- 远景：仅星系光团 + 黑洞
- 中景：恒星可见，行星轮廓出现
- 近景：行星细节、卫星环绕、标签星云、星座连线

### 5.2 大气突入阅读（核心体验）

点击行星后的 3 秒电影级动画：

1. **加速接近**（1s）——摄像机冲向行星
2. **穿透云层**（0.5s）——泛光 + 画面撕裂 + 色散特效
3. **微观降维**（0.5s）——行星表面溶解为 Voronoi 细胞网格
4. **着陆阅读**——2D 文章面板嵌入 3D 场景

**阅读面板设计**：

- 居中，占屏幕 70% 宽度
- 毛玻璃半透明背景，行星在背后缓慢自转（15fps 降速）
- 右侧浮动目录（悬停展开）
- 支持 Markdown 扩展：数学公式、流程图、代码沙盒、提示块
- 代码高亮：One Dark Pro 深色主题
- 阅读进度条 + 记忆上次位置

**退出**：

- ESC / 返回按钮 → 反向起飞动画（从行星表面升空回到轨道视角）
- 大气突入中断处理（浏览器返回）：立即中断动画

### 5.3 评论卫星发射

**发射仪式**：

1. 按 Enter → 飞船 HUD 终端面板滑出（等宽字体 + 扫描线）
2. 输入评论（支持完整 Markdown + 代码高亮）
3. 点击 **"发射 (Launch)"**
4. HUD 收缩为光束 → 沿曲线飞向行星 → 被引力捕获 → 凝结为新卫星

**评论者身份**：昵称（必填）+ 邮箱（必填，用于防冒名绑定 + 通知）+ Cookie 记忆
**太空头像编辑器**：用户可选组件拼装头像——底色（星云色调）+ 头盔样式（5-6 种）+ 面罩颜色 + 徽章（可选），存储为 JSON 配置，前端实时渲染 SVG
**评论者卡片**：点击头像弹出卡片（头像 + 昵称 + 评论数 + 最近评论），不做独立主页

**评论 placeholder 文案**：每篇文章不同，宇宙主题随机选取：

- "留下你的星际通讯..."
- "发射你的想法..."
- "向这颗星球投射信号..."

**热度可视化**：

- 少量评论：几条清晰的轨道环
- 大量评论（>50）：壮观的"土星环"效果
- 评论减少后卫星环直接反映当前状态，无过渡动画

**评论能力**：

- 嵌套回复：3 层，超出后平铺 @提及
- 排序：按时间/最热，用户可切换
- 分页：支持
- Markdown：完整语法（需严格 XSS 净化）
- 表情/Emoji：支持
- 评论点赞 / 举报 / 编辑（自己的）/ 删除（自己的）：支持
- 审核：需要审核，未通过前不可见
- WebSocket 实时推送评论（仅评论实时，其他不实时）

### 5.4 虫洞时间旅行

**虫洞入口**：场景边缘固定位置，环状结构

**虫洞穿越**（约 1.5 秒）：

1. 虫洞旋涡效果（径向扭曲 + 速度线 + 色彩偏移）
2. 到达历史星区（色调变暗，行星纹理风化）

**年份星区内**：

- 按 12 个月份分扇区布局，行星按发布时间排列
- 区域内有迷你虫洞可直接跳到相邻年份，也可返回主视图
- HUD 月份选择器可快速定位

## 六、视觉美学系统

### 6.1 配色体系

| 色彩名        | 色值     | 用途                              |
| ------------- | -------- | --------------------------------- |
| cosmic-void   | ~#0a0a1a | 深空背景（近真实太空色，非纯黑）  |
| cosmic-glow   | #38bdf8  | 主色调，脉冲蓝/青，HUD 边框/高亮  |
| cosmic-nebula | #6b21a8  | 辅助色，星云紫/品红               |
| cosmic-plasma | #fb923c  | 强调色，等离子橙（警告/行动按钮） |
| cosmic-frost  | #e2e8f0  | 冰冻白/浅灰，正文文字             |
| cosmic-crt    | #00d4ff  | CRT 终端冰蓝色                    |

**整体方向**：冷蓝/青色系 + 紫色/品红系混搭（赛博朋克科幻感）

### 6.2 字体

| 用途       | 字体                   | 风格                               |
| ---------- | ---------------------- | ---------------------------------- |
| 中文正文   | 霞鹜文楷 (LXGW WenKai) | 书法韵味，适合人文与技术并重的博客 |
| 西文正文   | Inter                  | 清晰的无衬线                       |
| 代码 / HUD | JetBrains Mono         | 等宽，终端感                       |
| 标题       | Space Grotesk          | 科幻几何感                         |

### 6.3 HUD 设计语言

- **多主题 HUD**：不同场景使用不同风格——军事/战术风、极简科幻风、复古 CRT 风、生物科技风
- **边框**：点线（dotted）+ 发光效果（glow/neon）
- **面板动画**：粒子凝聚——粒子从四周聚合成面板形状
- **面板透明度**：半透明毛玻璃——能看到背后的星空，但信息清晰
- **扫描线**：轻微（隐约可见，不影响阅读）

### 6.4 加载页面

**粒子凝聚风**：

- 散乱的星尘粒子逐渐聚合成一个球体
- 进度条环绕球体
- 阶段提示融入宇宙主题："校准引力透镜..."、"初始化物理引擎..."

**Logo 位置**：预留居中位置，名称 "CyberGeek"

### 6.5 404 页面

每次访问不存在的页面，随机展示三种风格之一：

- A) 迷失在太空中的漂浮宇航员
- B) 坐标不存在的虚空区域（黑暗 + "未知坐标"文字）
- C) 被黑洞吞噬的画面

### 6.6 低端设备处理

不做 2D 降级。低端设备（WebGL 不支持或最低画质仍跑不动）直接提示"设备不支持"。

## 七、内容驱动的视觉映射

### 7.1 行星外观与文章属性的关联

文章不是随机分配外观——每颗行星的视觉特征都编码了文章的内容属性。

| 文章属性   | 映射到行星视觉  | 规则                                                                        |
| ---------- | --------------- | --------------------------------------------------------------------------- |
| 字数       | 行星类型 + 体积 | < 1000 字 → 岩石行星（小）；1000~5000 → 类地行星；> 5000 → 气态巨行星（大） |
| 标签/分类  | 主色调          | 技术 → 冷蓝紫；人文 → 暖琥珀；安全 → 深红                                   |
| 代码块占比 | 表面纹理        | 代码密集 → 电路/网格纹理；纯叙事 → 有机地貌纹理                             |
| 评论数     | 卫星环          | 少评论 → 孤独行星；多评论 → 壮观行星环                                      |
| 文章年龄   | 表面粗糙度      | 新文章 → 光滑；老文章 → 风化纹理                                            |
| 近期活跃度 | 发光强度        | 有新评论 → 大气层活跃；长期无访问 → 暗淡                                    |
| 教程/系列  | 有无行星环      | 教程类文章 → 附带行星环（暗示丰富的附属内容）                               |

### 7.2 星座关联（相关文章连线）

内容相关的行星之间用发光连线形成"星座"图案：

- **关联度计算**：标签重合为主，pgvector 语义相似为辅
- **计算时机**：文章发布/更新时异步计算
- **视觉表现**：悬停行星时显示该文章的关联连线，平时不显示（常驻连线太杂乱）
- **关联度阈值**：固定阈值，不可调
- 连线亮度反映关联强度

### 7.3 搜索 3D 可视化

搜索不是弹出一个列表——搜索结果映射到 3D 空间：

- 匹配的行星脉冲发光，未命中天体灰显（透明度 15%）
- 所有命中天体间绘制微弱连线（"搜索星图"）
- 摄像机自动飞向命中天体的质心，调整视野使所有结果可见
- 侧边 HUD 同时显示文字列表

## 九、用户路由与导航

### 9.1 URL 映射

| URL               | 场景                           |
| ----------------- | ------------------------------ |
| `/`               | 宇宙全景（远景俯瞰）           |
| `/galaxy/[slug]`  | 飞向对应星系，聚焦中心黑洞     |
| `/post/[slug]`    | 大气突入，进入文章阅读         |

## 十二、性能与体验的平衡

### 12.1 自适应画质（3 级）

| 等级 | 策略                          | 触发条件         |
| ---- | ----------------------------- | ---------------- |
| 高   | 全特效 + 全粒子 + 精细 Shader | 默认             |
| 中   | 关闭部分后处理、粒子减半      | FPS < 30 持续 3s |
| 低   | 关闭所有后处理、粒子 1/4      | 连续降级         |

- 升级条件：FPS > 55 持续 10s
- 画质记忆：localStorage 记住上次等级
- DPR 上限：1.5（高分辨率屏幕）
- 低端设备（最低画质仍跑不动）：直接提示"设备不支持"，不做 2D 降级

### 12.2 渐进式场景揭示

不是一次加载所有天体——分 5 批按优先级加载：

1. **0-500ms**：深空背景 + 最近 2-3 个星系光团
2. **500ms-1.5s**：所有星系升级为粒子级 + 脉冲星闪烁
3. **1.5s-3s**：附近星系内行星渲染 + 射线检测激活
4. **3s-10s**：视野外星系逐个加载 + Shader 预热
5. **10s+**：小行星带填充 + 星座连线 + 非关键特效

用户交互时暂停后台加载（帧率优先）

### 12.3 仅暗色模式

宇宙天然是暗色的——不提供亮色模式切换，保持沉浸感统一。
