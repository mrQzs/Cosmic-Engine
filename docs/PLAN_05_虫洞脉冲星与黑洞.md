## Phase 8: 虫洞时间轴

### 8.1 虫洞 3D 模型与引力透镜效果
- **任务**：虫洞使用 `TorusGeometry` + 自定义 Shader 实现事件视界的扭曲效果——对背景星空的 `envMap` 进行 UV 扭曲采样，模拟引力透镜；虫洞中心发光、边缘星光弯曲
- **技术**：Three.js TorusGeometry、GLSL UV 扭曲 Shader、envMap 采样
- **产出**：`components/canvas/Wormhole.tsx`，视觉震撼的虫洞入口

### 8.1.1 虫洞空间排列与布局
- **任务**：定义虫洞在 3D 空间中的排列方式，形成视觉上可读的时间走廊：
  - 布局方案——时间螺旋走廊：虫洞沿一条螺旋曲线（`CatmullRomCurve3`）排列，最近的年份离当前星区最近，越久远的年份越远越深
  - 螺旋参数：每个年份间隔固定弧长，螺旋半径随年份递增（越老越远离中心），形成向外扩展的时间隧道
  - 虫洞朝向：每个虫洞的环面（Torus）法线指向螺旋切线方向，正面朝向摄像机来路
  - 年份标注：虫洞旁使用 `<Html>` Billboard 显示年份数字（发光等宽字体），颜色由近到远从亮蓝渐变为暗紫
  - 连接线：相邻虫洞之间绘制微弱的发光引导线（`Line2`），形成可追踪的时间路径
  - 入口虫洞：当前星区设置一个特殊的"时间走廊入口"虫洞（标注"归档 Archives"），摄像机从此处进入走廊
- **技术**：Three.js CatmullRomCurve3、Torus 朝向计算（`lookAt`）、R3F Html Billboard
- **产出**：`components/canvas/WormholeCorridor.tsx` 虫洞走廊布局组件；`utils/timelineCurve.ts` 螺旋曲线生成工具

### 8.1.2 虫洞预览信息
- **任务**：在进入虫洞前展示该年份的概要信息，降低盲跳成本：
  - 鼠标悬停虫洞时，虫洞边缘扩展出半透明 HUD 信息卡：
    - 年份（大字）
    - 文章数量（"12 颗行星"）
    - 评论总数（"348 颗卫星"）
    - 热门文章 Top 3 标题列表（可点击直接跳转到该文章，跳过走廊浏览）
    - 该年份最活跃的星系/分类
  - 悬停时虫洞视觉增强：扭曲效果加剧、中心发光增强、隐约显示目标星区的缩略景象（虫洞中心渲染目标星区的低分辨率快照）
  - 数据来源：后端 GraphQL 查询 `archivePreview(year: Int!)` 返回聚合摘要
- **技术**：R3F Html 组件、GraphQL 聚合查询、GLSL 虫洞中心纹理切换
- **产出**：`components/ui/WormholePreview.tsx` 虫洞预览卡片；后端归档预览查询

### 8.2 光速跃迁动画
- **任务**：点击虫洞后触发跃迁序列：① 摄像机加速冲向虫洞中心 ② 进入后屏幕拉伸星线（径向模糊 Shader 或 GSAP 驱动星空粒子拉丝）③ 短暂白屏闪烁 ④ 到达目标年份星区
- **技术**：GSAP timeline、自定义径向模糊 Shader、PostProcessing
- **产出**：`animations/wormholeJump.ts`，完整的虫洞跃迁转场效果

### 8.3 古代文明遗迹星区
- **任务**：跃迁目标星区的行星使用特殊材质——风化岩石纹理（降低饱和度、增加噪声纹理粗糙度）；星区整体色调偏暗偏冷（后处理 ColorCorrection 滤镜）；UI 标注年份
- **技术**：Three.js 材质参数调整、PostProcessing ColorCorrection
- **产出**：视觉上明显区别于"现代"星区的归档区域

### 8.4 时间轴数据与导航
- **任务**：后端按年份/月份聚合文章，每个年份对应一个虫洞实例；前端渲染虫洞列表并标注年份；支持在不同年份星区间跳转
- **技术**：GraphQL 聚合查询、前端状态管理
- **产出**：完整的按年份归档浏览功能

### 8.5 年内月份导航
- **任务**：跃迁到某年份星区后，提供月份级别的细粒度浏览：
  - 月份环带：年份星区内的行星按发布月份分布在 12 个扇区（30° 一个扇区），每个扇区对应一个月份
  - 月份标注：扇区边界绘制微弱分隔线，标注月份数字（1-12），形成类似钟表盘面的布局
  - 月份快速跳转：HUD 显示月份选择器（环形或横向滚动条），点击某月摄像机自动转向对应扇区
  - 空月份处理：无文章的月份扇区显示为暗淡区域，不生成分隔线，避免视觉噪音
  - 月份统计：选择器中每个月份旁显示文章数（小圆点或数字）

- **技术**：极坐标扇区划分、GSAP 摄像机旋转动画、React HUD 月份选择器
- **产出**：`components/ui/MonthSelector.tsx` 月份导航组件；年份星区的扇区布局逻辑

### 8.6 返回现代星区
- **任务**：在归档年份星区中提供清晰的返回路径：
  - 返回虫洞：每个年份星区中心放置一个特殊的"返回虫洞"（视觉区别：蓝白色调、反向旋转、标注"返回当前 ↩"），点击后触发反向跃迁动画回到现代星区
  - HUD 返回按钮：屏幕左上角常驻"返回当前星区"按钮，点击直接跳转（简化跃迁动画，缩短过渡时间）
  - 面包屑更新：在归档星区时面包屑显示 `宇宙 > 归档 > 2024年`，点击"宇宙"直接回到全景
  - 相邻年份跳转：HUD 显示 `← 2023` 和 `2025 →` 快速切换按钮，无需回到走廊再选择
  - 键盘快捷键：`Backspace` 返回上一个位置（走廊或现代星区）
- **技术**：Zustand 导航栈（历史记录）、GSAP 反向跃迁动画、React HUD 导航
- **产出**：归档星区的完整导航回路；`stores/navigationStore.ts` 导航历史栈

## Phase 9: 脉冲星个人简介

### 9.1 脉冲星 3D 模型
- **任务**：创建位于宇宙中心的脉冲星——高速自转的球体 + 两极射出的锥形射线束（`ConeGeometry` + 发光材质）；射线束周期性闪烁（`emissiveIntensity` 正弦波动）；强 `UnrealBloom` 泛光效果
- **技术**：Three.js 几何体组合、GLSL emissive 动画、Post-processing Bloom
- **产出**：`components/canvas/Pulsar.tsx`，宇宙中心耀眼的脉冲星航标

### 9.2 全息扫描展开动效
- **任务**：点击脉冲星后，触发"全息扫描"：① 脉冲星射线束收拢 ② 从星体表面向外扩展一圈全息扫描光环（`RingGeometry` 动画扩张）③ 扫描完成后展开个人简介面板
- **技术**：GSAP、Three.js 动画、R3F Html
- **产出**：`animations/pulsarScan.ts`，全息扫描展开动画

### 9.3 极客简介面板
- **任务**：个人简介不用传统排版，而是：① 雷达图展示技术栈熟练度（SVG 或 Canvas 雷达图）② 终端风格跳动的数据流（CSS 打字机效果）③ 时间线展示经历（发光节点连线）
- **技术**：React、SVG/Canvas 图表、CSS 动画
- **产出**：`components/ui/PulsarProfile.tsx`，硬核极客风个人简介

### 9.4 简介数据管理
- **任务**：为个人简介提供可维护的数据来源，避免硬编码：
  - 后端数据模型：`profile` 表或 `celestial_bodies` 中 `type = 'PULSAR'` 的记录，存储以下 JSONB 结构：
    - `bio`：个人简介文本（支持 Markdown）
    - `skills`：技术栈数组 `[{ "name": "Go", "level": 90, "category": "后端" }, ...]`，level 0-100 映射到雷达图
    - `timeline`：经历时间线 `[{ "year": "2020", "title": "xxx", "description": "..." }, ...]`
    - `social_links`：社交链接 `[{ "platform": "github", "url": "...", "icon": "..." }, ...]`
    - `stats`：个人统计（从其他表自动聚合：总文章数、总字数、博客运行天数、总评论数）
  - 管理后台编辑：专门的"脉冲星配置"页面，表单式编辑技能列表和时间线，实时预览雷达图效果
  - GraphQL 查询：`query { profile { bio, skills, timeline, socialLinks, stats } }`
- **技术**：PostgreSQL JSONB、Go 管理 API、React 管理后台表单
- **产出**：`backend/internal/profile/` 简介数据模块；管理后台简介编辑页面

### 9.5 联系方式与社交链接
- **任务**：在脉冲星简介面板中展示联系方式，融入太空主题：
  - 社交链接渲染为围绕简介面板的"通信卫星"——每个链接是一个小型发光图标球体，沿简介面板边缘缓慢环绕
  - 悬停卫星时放大显示平台名称和链接预览
  - 点击在新标签页打开对应链接
  - 支持的平台：GitHub、Twitter/X、邮箱（mailto:）、LinkedIn、个人网站、微信（显示二维码弹窗）、Telegram、RSS（链接到 Feed 页）
  - 邮箱防爬：邮箱地址不直接渲染为文本，使用 SVG 或 JS 动态拼接，防止被爬虫抓取
  - 简介面板底部："联系舰桥"区域集中展示所有社交图标（fallback 平面布局）
- **技术**：Three.js Sprite（通信卫星）、React 图标组件、mailto 链接、二维码生成
- **产出**：`components/canvas/SocialSatellites.tsx` 社交链接卫星；`components/ui/ContactBridge.tsx` 联系方式区域

## Phase 10: 星系中心超大质量黑洞 (Galaxy Core SMBH)

> **概念变更**：黑洞不再映射"加密文章"（项目不需要加密功能），而是回归其天体物理学本源——每个星系（主分类）中心的超大质量黑洞（Supermassive Black Hole），作为星系的引力锚点和分类概览入口。同时引入"恒星"概念表示子分类。

### 10.1 星系中心黑洞 Shader
- **任务**：每个星系中心放置一个超大质量黑洞。黑洞无显式 3D 模型——在不可见球体上编写 **Raymarching 光线步进** Shader；采样背后环境星空贴图 `envMap`，依据施瓦西度规公式扭曲 UV 坐标，模拟光线弯曲；吸积盘使用旋转的 `RingGeometry` + 发光粒子
  - 黑洞视觉参数与星系数据联动：
    - 吸积盘颜色 = 星系 `color_scheme` 主色调
    - 黑洞视觉半径 ∝ 星系总文章数（文章越多，黑洞引力越强）
    - 吸积盘旋转速度 ∝ 近期发布频率（活跃的星系吸积盘转得更快）
- **技术**：GLSL Raymarching、UV 扭曲、envMap 采样、施瓦西度规简化
- **产出**：`shaders/blackHole.glsl`、`components/canvas/GalaxyCoreBlackHole.tsx`

### 10.2 黑洞分类概览面板
- **任务**：点击星系中心黑洞后，展示该分类的概览信息（替代原来的 CRT 加密终端）：
  - 全息投影式概览面板在吸积盘上方展开（R3F `<Html>` 组件 + HUD 风格）
  - 概览内容：
    - 该星系（主分类）名称和描述
    - 恒星（子分类）列表及各子分类文章数量
    - 热门文章 Top 5（阅读量排序）
    - 最近发布的 5 篇文章
    - 总文章数、总评论数统计
  - 点击恒星名称 → 摄像机飞向对应恒星
  - 点击文章标题 → 摄像机飞向对应行星 → 大气突入
- **技术**：R3F Html 组件、GraphQL 聚合查询、GSAP 展开动画
- **产出**：`components/ui/GalaxyOverview.tsx`，星系概览 HUD 面板

### 10.3 恒星（Star）渲染与生命周期
- **任务**：创建 `<Star>` 组件，表示子分类。恒星的外观根据其包含的文章数量动态变化，呈现从"原恒星盘"到"红巨星"的完整恒星生命周期：
  - **数据来源**：`galaxies` 表中 `parent_id IS NOT NULL` 的记录即为恒星（子分类），其 `article_count` 字段（后端自动维护）决定生命周期阶段
  - **四阶段视觉**：
    | 阶段 | 条件 | 3D 表现 |
    |------|------|---------|
    | 原恒星盘 | article_count < 10 | 暗淡的旋转气体云团（`Points` 粒子 + Curl Noise 驱动），不发光，行星在云雾中若隐若现 |
    | 主序星 | article_count ≥ 10 | 明亮的自发光球体（`SphereGeometry` + 高 `emissive`），暖黄色，表面 Simplex Noise 纹理模拟对流，有日冕光晕（Fresnel + Bloom） |
    | 巨星 | article_count ≥ 50 | 体积 ×1.5，颜色偏橙，表面纹理更剧烈，光晕范围扩大 |
    | 红巨星 | article_count ≥ 100 | 体积 ×2.5，深红色，表面有明显的对流湍流动画，抛射星风粒子效果（径向扩散的半透明粒子） |
  - **恒星物理参数**：
    - 恒星围绕星系中心黑洞公转（开普勒椭圆轨道，复用 PLAN_03 §4.4 逻辑）
    - 恒星轨道半径 = 星系半径 × 0.3~0.7（由后端 `galaxies.position` JSONB 中的 `orbit_radius` 决定）
    - 行星围绕恒星公转（而非围绕星系中心），轨道半径相对恒星更小（20~100 单位）
  - **恒星材质**：使用 `ShaderMaterial` 实现表面对流纹理（多层叠加的 Simplex Noise，随时间演化），颜色随生命周期阶段从黄白 → 黄 → 橙 → 深红渐变
- **技术**：Three.js SphereGeometry + ShaderMaterial、Curl Noise 粒子（原恒星盘）、Fresnel + Bloom（日冕）、GLSL Simplex Noise（表面对流）
- **产出**：`components/canvas/Star.tsx`，恒星组件；`shaders/starSurface.glsl`，恒星表面着色器

### 10.3.1 恒星诞生动画 (Star Ignition)
- **任务**：当子分类的文章数从 9 增长到 10 时，触发"恒星点燃"动画——这是整个博客中最壮观的自然事件之一：
  - **动画序列**（GSAP timeline，总时长 3 秒）：
    - ① 原恒星盘的粒子开始加速旋转、向中心收缩（0~1s）
    - ② 粒子密度急剧增高，中心开始发出微弱白光（1~1.5s）
    - ③ 临界点——中心白光爆发，冲击波向四周扩散（环形 `RingGeometry` 快速放大 + 淡出），伴随强烈的 Bloom 泛光（1.5~2s）
    - ④ 残余云雾被冲击波吹散，一颗明亮的主序星出现在原位（2~3s）
    - ⑤ 此前在云雾中隐约可见的行星，现在清晰地围绕新生恒星公转
  - **触发条件**：后端 API 检测到 `galaxies.article_count` 从 < 10 变为 ≥ 10 时，通过 WebSocket 推送 `star_ignited` 事件；前端收到后播放动画
  - **音效**：低频轰鸣 → 能量聚集高频声 → 爆发冲击音 → 恒星持续低频嗡鸣
  - **性能**：动画仅在当前视野内的恒星触发；如果用户不在该星系视野内，下次进入时直接显示主序星状态（跳过动画）
- **技术**：GSAP timeline、Three.js 粒子系统 + RingGeometry（冲击波）、PostProcessing Bloom 动态强度、WebSocket 事件
- **产出**：`animations/starIgnition.ts`，恒星诞生动画函数

### 10.3.2 恒星交互与子分类概览
- **任务**：点击恒星显示该子分类的信息和文章列表：
  - 摄像机平滑飞向恒星（GSAP flyTo）
  - 恒星旁展开 HUD 面板（全息投影风格）：
    - 子分类名称和描述
    - 文章列表（按发布时间倒排，可滚动）
    - 文章数量 / 评论总数统计
    - "恒星光谱"信息条：用颜色编码展示该子分类的生命周期阶段和活跃度
  - 点击面板中的文章标题 → 摄像机飞向对应行星 → 大气突入
  - ESC 或点击空白处 → 摄像机回退到星系全景
- **技术**：R3F Html 组件、GSAP 动画、GraphQL 子分类查询
- **产出**：`components/ui/StarOverview.tsx`，子分类概览面板

### 10.4 多星系黑洞视觉差异化
- **任务**：每个星系都有自己的中心黑洞，它们的视觉外观应有差异以便识别：
  - 吸积盘颜色：跟随星系 `color_scheme` 主色调——杂志星系蓝紫色、小说星系暖琥珀色等
  - 黑洞视觉大小 ∝ 星系总文章数——文章越多的星系，中心黑洞越壮观
  - 吸积盘活跃度 ∝ 近期发布频率——活跃的星系吸积盘旋转更快、粒子更密集；沉寂的星系吸积盘缓慢旋转
  - 标识：每个黑洞旁用 `<Html>` Billboard 显示星系（主分类）名称
  - 引力场范围差异：大黑洞的引力透镜扭曲范围更广，视觉上影响更远距离的背景星空
- **技术**：Three.js 材质参数 seed 化、GLSL uniform 变量控制吸积盘颜色和速度、InstancedMesh（如黑洞数量多）
- **产出**：黑洞组件的 seed 参数化外观系统

### 10.5 访客统计星图
- **任务**：在脉冲星个人简介旁渲染一个微型"访客星系"：
  - 后端记录访客 IP 并通过 GeoIP 数据库（MaxMind GeoLite2）解析为地理坐标
  - 每个访客来源城市/国家渲染为一颗微小光点，位置映射到球面坐标
  - 访问量越多的地区光点越亮、越大，形成自然的热力分布

## Phase 8-10 补充: Shader 性能、场景切换内存管理与过渡流畅度

### 8.7 虫洞跃迁场景切换的内存管理

- **任务**：虫洞跃迁从"现代星区"到"归档星区"时，需要妥善管理旧场景的 GPU 资源释放和新场景的加载，防止内存累积：
  - **问题说明**：跃迁到 2023 归档星区后，现代星区的行星纹理、材质、粒子仍占用 GPU 显存。多次来回跃迁会导致显存持续增长（Three.js 不自动释放 GPU 资源）
  - **分级管理策略**：
    | 距离级别 | 资源处理 | 说明 |
    |----------|----------|------|
    | 当前星区 | 完整加载 | 所有天体精细渲染 |
    | 相邻星区（±1 年） | 保留轻量数据 | 仅保留位置坐标和 LOD 最低级（光点），dispose 所有纹理和精细几何体 |
    | 远距离星区（>±2 年） | 完全卸载 | dispose 所有 GPU 资源，仅在 Zustand 中保留元数据 |
  - **跃迁时的生命周期**：
    ```
    ① 跃迁动画开始
    ② 动画进行中（星线拉丝阶段）：后台 async 加载目标星区资源
    ③ 动画结束、新星区渲染就绪
    ④ 延迟 2 秒后：dispose 旧星区的精细资源（避免用户立即返回时重复加载）
    ⑤ 如果 5 秒内用户返回：从内存缓存恢复（跳过网络加载）
    ⑥ 超过 5 秒：旧资源彻底释放
    ```
  - **显存监控**：遵循 Phase 1 §1.18 规范，跃迁前后对比 `renderer.info.memory`，确认资源释放成功
- **技术**：Three.js dispose()、延迟释放（setTimeout + WeakRef 缓存）、`renderer.info.memory`
- **产出**：`hooks/useSceneTransition.ts` 场景切换资源管理 Hook

### 10.5.1 黑洞 Raymarching Shader 性能优化

- **任务**：Raymarching Shader 是项目中计算最重的着色器，需要专项优化以保证可用帧率：
  - **性能分析**：黑洞 Shader 的 Fragment Shader 中，每个像素执行 50-200 次光线步进循环。在 1920×1080 分辨率下 = 约 200 万像素 × 100 次循环 = 2 亿次计算/帧
  - **优化策略**：
    - **步进次数自适应**：根据画质等级调整最大步进次数：高 = 128、中 = 64、低 = 32。通过 uniform 传入，无需重新编译 Shader
    - **距离衰减步长**：光线离黑洞越远，步长越大（`stepSize *= 1.05`），减少远距离的无效计算
    - **渲染区域限制**：黑洞 Shader 仅在一个有限半径的球体 Mesh 上运行，球体外不执行 Raymarching
    - **分辨率降级**：黑洞 Mesh 使用独立的低分辨率 RenderTarget 渲染（0.5x 主分辨率），再贴回主场景。视觉差异极小（黑洞边缘本身就是模糊的），但像素计算量减少 75%
    - **多黑洞场景**：仅渲染摄像机最近的 2 个黑洞的完整 Raymarching，其余使用简化的 UV 扭曲 Shader（无步进循环）
  - **Shader 预热**：黑洞 Shader 必须在 Phase 1 §1.20 的预热阶段编译完成，避免用户首次看到黑洞时的帧冻结
  - **移动端策略**：移动端直接使用简化 UV 扭曲 Shader 替代 Raymarching（画质等级"低"以下自动切换）
- **技术**：GLSL uniform 画质控制、RenderTarget 降分辨率渲染、Shader LOD
- **产出**：`shaders/blackHole.glsl` 中的自适应步进逻辑；`shaders/blackHoleSimplified.glsl` 简化版 Shader

### 10.5.2 HUD 面板与 3D 场景叠加的流畅度保障

- **任务**：星系中心黑洞概览面板（§10.2）和恒星概览面板（§10.3.2）的弹出/收起涉及 DOM 层与 WebGL 层叠加渲染，需确保过渡期间零卡顿：
  - **3D 降速策略**：HUD 面板展开时，将 3D 渲染帧率从 60fps 降至 15fps（减少 GPU 负载，让 CSS 动画流畅）：
    ```tsx
    // 面板打开时
    gl.setAnimationLoop(null) // 暂停自动渲染循环
    const slowLoop = setInterval(() => gl.render(scene, camera), 66) // 手动 15fps
    // 面板关闭时恢复
    clearInterval(slowLoop)
    gl.setAnimationLoop(renderLoop)
    ```
  - **粒子凝聚面板动画**（§4.3 HUD 动画偏好 D）：粒子从四周聚合成面板形状——使用 CSS `transform` + `will-change` 实现，不走 WebGL 管线
  - **帧率保护**：过渡期间如果 FPS < 40，跳过粒子凝聚动画，直接 fade 弹出
- **技术**：WebGL 渲染循环手动控制、CSS `will-change`、`requestAnimationFrame` 时序
- **产出**：HUD 面板过渡的帧率保护逻辑；WebGL 降速/恢复工具函数
